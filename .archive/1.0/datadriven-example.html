data-ui-event Event The event on which to trigger the accordion.

Class / Data Attribute Values Description
class="autocomplete disabled" disabled Disables (true) or enables (false) the autocomplete. Can be set when initialising (first creating) the autocomplete.

data-ui-autocomplete-position-at Defines which position on the target element to align the positioned element against: "horizontal vertical" alignment. A single value such as "right" will default to "right center", "top" will default to "center top" (following CSS convention).
data-ui-autocomplete-position-collision "flip", "fit", "fit flip", "fit none" When the positioned element overflows the window in some direction, move it to an alternative position. Similar to my and at, this accepts a single value or a pair.

data-ui-datepicker-maxdate Date Value, Restricted Days (optional)

The maximum selectable date.

Date Value
  • Date: A date string containing the maximum date. For example, "01/01/2013" would set the max date to January 1, 2013.
  • Integer: A number of days from today. For example 2 represents two days from today and -1 represents yesterday.
  • String: A string in the format defined by the dateFormat option, or a relative date. Relative dates must contain value and period pairs; valid periods are "y" for years, "m" for months, "w" for weeks, and "d" for days. For example, "+1m +7d" represents one month and seven days from today.
Restricted Days
  • true - does not include any weekends or holidays in your max day count. For example, if you have data-ui-datepicker-noweekends="true" set, a max date value of 7 will actually be 9 days from today, since weekends aren't counted.
  • false - includes any weekends or holidays in your max day count. For example, if you have data-ui-datepicker-noweekends="true" set, a max date value of 7 will actually be 7 days from today, since weekends are still counted.

NOTE: The restricted days only work with an Integer Date Value. It does not work with the Date or String variations.

Responsive Tables

The responsive tables mode selectively hides columns at narrower widths as a sensible default but also offers a menu to let users manually control which columns they want to see. The table works by hiding and showing columns based on two inputs: available screen width or by the user checking and unchecking which columns to display in a column picker popup.

Responsive Table Options using the Data-Driven plugin
Class / Data Attribute Values Description
id String An ID attribute is also required on the table to associate it with the column chooser popup menu. This will also assist in the creation of responsive CSS rules.
data-role table Applies the responsive table functionality to the table.
data-priority Number

Add data-priority attributes to each of the table headers of columns you want to responsively display and assign a priority (1 = highest, 6 = lowest). Any table header given a priority will be available in the column picker menu.

To make a column persistent so it's not available for hiding, omit the data-priority attribute. This will make the column visible at all widths and won't be available in the column chooser menu.

NOTE: You may use any priority naming convention and assign as many levels of priority for the columns. The plugin simply generates class names based on the values in the data-priority attribute. For example, if a priority of data-priority="critical" is added to the heading, a class of ui-table-priority-critical will be applied to each cell in that column.

data-ui-position-collision
  • "flip": Flips the element to the opposite side of the target and the collision detection is run again to see if it will fit. Whichever side allows more of the element to be visible will be used.
  • "fit": Shift the element away from the edge of the window.
  • "flipfit": First applies the flip logic, placing the element on whichever side allows more of the element to be visible. Then the fit logic is applied to ensure as much of the element is visible as possible.
  • "none": Does not apply any collision detection.

When the positioned element overflows the window in some direction, move it to an alternative position. Similar to my and at, this accepts a single value or a pair for horizontal/vertical, e.g., "flip", "fit", "fit flip", "fit none".

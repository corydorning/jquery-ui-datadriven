Data-driven jQuery Framework

The data-driven framework exposes many of the useful jQuery UI options using the HTML5 data-attribute. This allows you to customize widget instances right from the HTML instead of individual, custom scripts for each instance. It also provides a lot of valuable use cases that aren't provided with jQuery UI and saves you the time from writing them.

Autocomplete

Enables users to quickly find and select from a pre-populated list of values as they type, leveraging searching and filtering.

API

Autocomplete Options using the Data-Driven plugin

Attribute Values

data-role

Required
autocomplete

Creates the autocompleter.

data-ui-source

Required
Array

An array can be used for local data. There are two supported formats:

  • An array of strings: ["Choice1", "Choice2", ... ]
  • An array of objects with label and value properties: [ { label: "Choice1", value: "value1" }, ... ]. The label property is displayed in the suggestion menu. The value will be inserted into the input element when a user selects an item. If just one property is specified, it will be used for both, e.g., if you provide only value properties, the value will also be used as the label.
Function

A callback function provides the most flexibility and can be used to connect any data source to Autocomplete. The callback gets two arguments:

  • A request object, with a single term property, which refers to the value currently in the text input. For example, if the user enters "new yo" in a city field, the Autocomplete term will equal "new yo".
  • A response callback, which expects a single argument: the data to suggest to the user. This data should be filtered based on the provided term, and can be in any of the formats described above for simple local data. It's important when providing a custom source callback to handle errors during the request. You must always call the response callback even if you encounter an error. This ensures that the widget always has the correct state.

String

When a string is used, the Autocomplete plugin expects that string to point to a URL resource that will return JSON data. It can be on the same host or on a different one (must provide JSONP). The Autocomplete plugin does not filter the results, instead a query string is added with a term field, which the server-side script should use for filtering the results. For example, if the source option is set to "http://example.com" and the user types foo, a GET request would be made to http://example.com?term=foo. The data itself can be in the same format as the local data described above.

class
disabled

Disables the autocomplete. (NOTE: This does not disable the textbox)

data-ui-autocomplete-autofocus
Boolean

If set to true the first item will be automatically focused.

data-ui-autocomplete-delay
Integer

The delay in milliseconds the Autocomplete waits after a keystroke to activate itself. A zero-delay makes sense for local data (more responsive), but can produce a lot of load for remote data, while being less responsive.

data-ui-autocomplete-minlength
Integer

The minimum number of characters a user has to type before the Autocomplete activates. Zero is useful for local data with just a few items. Should be increased when there are a lot of items, where a single character would match a few thousand items.

data-ui-position-my
"top", "center", "bottom", "left", "right"

Defines which position on the element being positioned to align with the target element: "horizontal vertical" alignment. A single value such as "right" will default to "right center", "top" will default to "center top" (following CSS convention).

Additional reading about jQuery UI Positioning.

data-ui-position-at
"top", "center", "bottom", "left", "right"

Defines which position on the element being positioned to align with the target element: "horizontal vertical" alignment. A single value such as "right" will default to "right center", "top" will default to "center top" (following CSS convention).

Additional reading about jQuery UI Positioning.

data-ui-position-collision
"flip", "fit", "fit flip", "fit none"

When the positioned element overflows the window in some direction, move it to an alternative position. Similar to my and at, this accepts a single value or a pair.

Additional reading about jQuery UI Positioning.

Buttons & Buttonsets

Enhances standard form elements like buttons, inputs and anchors to themeable buttons with appropriate hover and active styles. data-ui-datepicker-changemonth
Boolean

If set to true, dropdown is displayed to change the month

data-ui-datepicker-defaultdate
Date

A date string containing the minimum date. For example, "01/01/2013" would set the min date to January 1, 2013.

Number

A number of days from today. For example 2 represents two days from today and -1 represents yesterday.

Relative Date

Relative dates must contain value and period pairs; valid periods are "y" for years, "m" for months, "w" for weeks, and "d" for days. For example, "+1m +7d" represents one month and seven days from today.

data-ui-datepicker-maxdate

The maximum selectable date. Consists of comma delimited Date Value and optional Restricted Date parameter.

Date Value

Date

Integer

A number of days from today. For example 2 represents two days from today and -1 represents yesterday.

Relative Date

Relative dates must contain value and period pairs; valid periods are "y" for years, "m" for months, "w" for weeks, and "d" for days. For example, "+1m +7d" represents one month and seven days from today.

Restricted Days (optional)

NOTE: The restricted days parameter only works with an Integer Date Value. It does not work with the Date or Relative Date variations.

true

Does not include days that are not selectable in your max day count. For example, if you have data-ui-datepicker-noweekends="true" set, a max date value of 7 will actually be 9 days from today, since weekends are disabled and would not be counted.

Days
